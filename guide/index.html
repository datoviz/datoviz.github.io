    Learn Datoviz¶

    Welcome to the Datoviz user guide. This section introduces the main concepts of the library, shows how to create and customize visualizations, and explains advanced usage patterns when you're ready to go deeper.

    What you'll learn¶

    This guide is structured into three levels:

    1. Essentials¶

    Start here if you're new to Datoviz:

    • 📌 Quickstart — create your first scatter plot in just a few lines of code
    • 📐 Main Concepts — learn how figures, panels, and axes structure your visual scene

    2. Visuals¶

    Each visual primitive has its own page with examples and documentation:

    • 🧭 Overview
    • 🔺 Basic — raw Vulkan primitives (points, line strips, triangles) with uniform color and no custom shaders
    • 🟦 Pixel — individual pixels with position, color, and size (squares)
    • ⚪ Point — sized, colored antialiased discs without borders
    • ✳️ Marker — symbols with optional borders, using built-in shapes, SVGs, or bitmap images
    • ➖ Segment — variable-width line segments with optional caps
    • ➰ Path — continuous polylines
    • 🖼️ Image — RGB or single-channel images with optional colormaps
    • 🔤 Glyph — glyph-based text rendering
    • 🧊 Mesh — 3D triangle meshes with optional lighting (flat, basic, or advanced), textures, contours, or experimental isolines
    • 🔮 Sphere — 3D spheres with customizable lighting and materials
    • 🌫️ Volume — basic volume rendering of dense 3D voxel data
    • 🪓 Slice — 2D orthogonal slices through 3D volumes

    3. Advanced Topics¶

    Explore these pages to customize behavior or build more complex apps:

    • 🖱️ Interactivity — built-in pan, zoom, and arcball camera controls
    • 🎮 Input — define keyboard and mouse event callbacks
    • ⏱️ Timers and Frame Events — run code every frame or at regular time intervals
    • 🧰 GUI Support (ImGui) — use Datoviz’s built-in ImGui layer for interactive widgets
    • 🔧 Datoviz Rendering Protocol (DRP) — internal low-level rendering architecture
    • ⚙️ Using Datoviz in C — native C interface for full performance and control

    Where to go next¶

    • Start with the Quickstart to render your first plot
    • Then read Main Concepts to understand how visuals are positioned and transformed
    • Or explore the Gallery to see what’s possible

    Need full technical details? Visit the API Reference.
